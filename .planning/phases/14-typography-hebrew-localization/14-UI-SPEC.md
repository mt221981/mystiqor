## Phase Scope

This phase has **no new UI screens**. It is a targeted typography and text correction pass across existing screens. The design contract governs:

1. **Font application** — Heebo must be the active font for all Hebrew body text across the app

- Colors, spacing, layout, component structure
- Font for headings (Plus Jakarta Sans on `--font-headline` is intentional and remains)
- Font for labels (Manrope on `--font-label` remains for tab/badge labels)
- Any backend, API, or database values (tool type slugs stay as `human_design`, `solar_return`, `drawing`)

### Phase 14 Typography Fix

**Problem identified:** `mystiqor-build/src/app/layout.tsx` line 112 sets `className="font-body"` on `<body>`. The `font-body` Tailwind token resolves to `['var(--font-hebrew)', 'Heebo', 'var(--font-body)', 'Inter', 'sans-serif']` (from `tailwind.config.ts` line 27). This is correct — Heebo loads first.

**Verification task:** Confirm `--font-hebrew` CSS variable resolves to the loaded Heebo font at runtime (not falling through to Inter). The `heebo` font object in layout.tsx sets `variable: '--font-hebrew'` — this is the correct pattern.

**Fix scope:** If any component uses `font-sans` (Tailwind default) instead of `font-body`, replace with `font-body`. No component should use bare `font-sans` for user-facing Hebrew text.

## Implementation Notes for Executor

1. **Heebo is already loaded.** `layout.tsx` imports `Heebo` from `next/font/google` and sets `variable: '--font-hebrew'`. The `font-body` Tailwind utility already includes Heebo as the first font. No new font installation needed.

2. **Confirm `font-body` propagates.** The `<body>` tag has `className="font-body"`. Verify this class cascades correctly to all text elements. Components that override with `font-sans` must be found and corrected.

3. **`tool-names.ts` is already correct.** `TOOL_NAMES` already maps `solar_return → 'חזרת שמש'` and `human_design → 'עיצוב אנושי'`. Do not touch this file.

4. **Replace display strings, not code.** Use a targeted grep for the three English terms in `.tsx` files. Review each match — only replace strings that appear in JSX return values (display text), not in type definitions, variable names, or comments.

5. **HTP replacement strategy.** "בית-עץ-אדם" is the full Hebrew name. In context where screen real estate is limited, "בית-עץ-אדם" may be used as-is (it is a well-known term in Hebrew psychology). Do not abbreviate to initials.

6. **No visual regression.** This phase must not change any layout, spacing, color, or component structure. Text replacements only, plus any `font-sans` → `font-body` corrections.

7. **Display weight.** All headline/display text uses `font-semibold` (600). Do not apply `font-bold` (700) to any element — it is not part of the declared weight scale.
