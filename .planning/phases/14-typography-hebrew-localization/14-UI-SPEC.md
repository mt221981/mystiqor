## Typography

### Role Definitions

| Role | Font Family | Size | Weight | Line Height | Usage |
|------|-------------|------|--------|-------------|-------|
| Display | Plus Jakarta Sans (`font-headline`) | 28px (text-2xl) | 700 | 1.2 | Page hero titles, onboarding step heads |
| Heading | Plus Jakarta Sans (`font-headline`) | 20px (text-xl) | 600 | 1.3 | Card titles, section heads, tool names |
| Body | Heebo (`font-body`) | 16px (text-base) | 400 | 1.7 | All Hebrew paragraph text, AI-generated insight copy |
| Label | Manrope (`font-label`) | 14px (text-sm) | 500 | 1.4 | Form labels, badge text, stat captions, nav items |

### Hebrew-Specific Typography Rules

1. **Primary Hebrew font is Heebo.** Every element that renders user-facing Hebrew text MUST use `font-body` (which resolves to `var(--font-hebrew)` → Heebo → Inter fallback). Do not use `font-headline` or `font-label` for Hebrew paragraph content.
2. **Line height for Hebrew body text is 1.7** (not 1.5). Hebrew letters have larger ascenders/descenders and optical crowding at 1.5. Use `leading-[1.7]` or set `line-height: 1.7` explicitly.
3. **No letter-spacing on Hebrew.** Never apply positive `letter-spacing` or Tailwind `tracking-*` utilities to Hebrew text. Hebrew script is sensitive to inter-character spacing and becomes unreadable with tracking. Use `tracking-normal` (0) as the explicit reset.
4. **RTL text alignment.** Hebrew text nodes use `text-start` (never `text-right` / `text-left`). The root layout already sets `dir="rtl"`.
5. **Font weight for Hebrew readability.** Hebrew at weight 300 is illegible at small sizes. Minimum weight for Hebrew body text is 400. Labels may use 500 but not 300.
6. **Numerals in Hebrew context.** Use `tabular-nums` for numeric values embedded in Hebrew text (e.g., gematria values, numerology scores) to prevent reflow.

### Font Fallback Chain

```
font-body (Hebrew primary):
  var(--font-hebrew) → Heebo → var(--font-body) → Inter → sans-serif

font-headline (Latin headings):
  var(--font-headline) → Plus Jakarta Sans → sans-serif

font-label (UI labels):
  var(--font-label) → Manrope → sans-serif
```

Source: `tailwind.config.ts` fontFamily definitions; Hebrew line-height is a localization default.

## Interaction Contracts

### Font Loading Behavior

- Use `font-display: swap` on all Google Font declarations (already set in `layout.tsx` — confirm preserved)
- During FOUT (Flash of Unstyled Text): body fallbacks to Inter/sans-serif; layout must not shift — ensure all Hebrew text containers have explicit `min-height` or `height` declarations to prevent CLS
- Heebo is loaded with `subsets: ['hebrew', 'latin']` — both subsets required; do not trim to latin-only

### RTL Layout Rules

- Root `dir="rtl"` is set on `<html>` in `layout.tsx` — do not override at component level
- Use logical properties throughout: `ps-` / `pe-` (padding-start / padding-end), `ms-` / `me-` (margin-start / margin-end), `text-start` / `text-end`
- Never use `text-left`, `text-right`, `pl-`, `pr-`, `ml-`, `mr-` in new components
- Flex direction: `flex-row` is visually reversed in RTL — confirm icon+text order in navigation and tool cards renders correctly (icon should appear on the right in Hebrew context)

### Accessibility (Hebrew-specific)

- All Hebrew text nodes must have `lang="he"` inherited from root `<html lang="he">` — do not override to `lang="en"` at component level unless the content is genuinely in English
- Screen reader support: Hebrew vowelization (nikud) is not used in UI copy — keep labels simple and unambiguous
- Minimum contrast: body text (`#ece6f5`) on dark background (`#0d0b1e`) = contrast ratio 14.5:1 — WCAG AAA
