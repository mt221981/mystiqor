**Step 2: Codebase-wide audit for remaining old colors**

Run grep to find ANY remaining old color patterns in .tsx files under src/:

```bash
grep -rn "text-gray-\|bg-gray-\|text-purple-\|bg-purple-\|text-indigo-\|bg-indigo-\|from-purple-\|to-purple-\|from-indigo-\|to-indigo-\|border-gray-\|border-white/" mystiqor-build/src/ --include="*.tsx" | grep -v "node_modules" | grep -v ".test."
```

For each match found:
1. Read the file
2. Apply the universal replacement rules
3. Preserve all logic

Common expected stragglers:
- `text-red-400` in error messages → replace with `text-error`
- `bg-white` in light mode fallbacks → OK to leave if in `:root` only
- `border-white/5` in minor components → `border-outline-variant/10`

Exceptions (do NOT change):
- Recharts `fill` or `stroke` props that already use hex values (#ddb8ff etc.)
- SVG internal colors (like birth chart components) that use specific astronomical colors
- Print styles in globals.css (`background: white`, `color: black` — these are correct for print)
