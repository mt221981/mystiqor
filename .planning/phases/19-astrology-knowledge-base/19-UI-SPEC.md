## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline badge padding |
| sm | 8px | Card inner content gaps (`gap-2`), tight row spacing |
| md | 16px | Default card padding (`p-4`), section gaps |
| lg | 24px | Card header padding (`pb-3` + `p-6`), tab content spacing |
| xl | 32px | Grid gaps between sections |
| 2xl | 48px | Major section vertical breaks on page |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions:
- Strength bar height: 6px (`h-1.5`) вАФ sub-4px exception inherited from `AspectList.tsx` pattern
- Touch targets on interactive cards: minimum 44px height (mobile a11y requirement)

Source: `PlanetTable.tsx` + `AspectList.tsx` established patterns.

## Color

All values from `tailwind.config.ts` MD3 tokens + `globals.css` dark mode CSS variables.

| Role | Token / Hex | Usage |
|------|-------------|-------|
| Dominant (60%) | `surface` `#0d0b1e` / `bg-background` | Page background (dark mode) |
| Secondary (30%) | `surface-container` `#181538` / `bg-surface-container` | All cards, panels, tab content areas |
| Accent (10%) | `gold` `#d4a853` / `accent` in dark mode | See reserved list below |
| Destructive | `destructive` `hsl(7 100% 84%)` / `#ffb4ab` | Destructive actions only (none in this phase) |

**Accent (`gold` #d4a853) is reserved ONLY for:**
- Ruler badge border + text on ZodiacGrid cards (`Badge` with `style={{ borderColor: sign.color, color: sign.color }}` вАФ when the sign's own color is used, not gold specifically)
- Page heading gradient (`text-gradient-gold` if PageHeader follows Phase 16 standard)
- Active tab indicator (shadcn Tabs active state)

**Per-entity colors (inline `style`):**
- Each zodiac sign: `sign.color` from `ZODIAC_SIGNS` in `astrology-data.ts` (12 distinct colors)
- Each planet: `planetInfo.color` from `PLANETS` in `astrology-data.ts` (10 distinct colors)
- Each aspect: `aspect.color` from `ASPECTS` in `astrology-data.ts` (7 distinct colors) вАФ used for Badge border, Badge text, strength bar fill
- These are data-driven colors, not design system additions

**Secondary surface for cards:** `bg-surface-container` with border `border-outline-variant/5`
**Tab bar background:** `bg-surface-container-high` (`#222048`)
**On-surface text:** `text-on-surface` (`#ece6f5`) for primary content, `text-on-surface-variant` (`#c8bede`) for secondary/metadata

Source: `tailwind.config.ts` MD3 tokens, `globals.css` dark mode variables, `AspectList.tsx` badge + bar pattern.

## Interaction Contract

| Interaction | Behavior | Implementation |
|-------------|----------|----------------|
| Tab switch | Instant content swap вАФ no loading state (data is static) | shadcn `Tabs` вАФ no async needed |
| House accordion expand | Single item open at a time (`type="single" collapsible`) | shadcn `Accordion` default |
| Card hover | `mystic-hover` lift + purple glow вАФ 0.3s ease | `.mystic-hover` CSS utility (globals.css) |
| Strength bar render | Static fill on mount вАФ no animation required | Inline `style.width` only |
| Page load | No skeleton вАФ data is synchronous static import | Direct render, no loading state |

**No mutations, no forms, no destructive actions in this phase.**

## Accessibility Contract

| Concern | Requirement | Implementation |
|---------|-------------|----------------|
| RTL layout | All containers `dir="rtl"` | `<Tabs dir="rtl">`, `<Accordion dir="rtl">`, grids inherit from root layout |
| Emoji | `aria-hidden="true"` on all decorative emoji | Matches `PlanetTable.tsx` pattern |
| Strength bar | `role="progressbar"` + `aria-valuenow` + `aria-valuemin` + `aria-valuemax` | Matches `AspectList.tsx` pattern |
| Color-only info | Aspect badge has both color AND text label | Badge shows name, not color alone |
| Card interactivity | Cards with `mystic-hover` are presentational only вАФ no `role="button"` unless clickable | Cards in this phase are read-only reference |
| Accordion keyboard | shadcn Accordion is fully keyboard navigable | Built-in Radix behavior |
| Focus ring | `ring-ring` from CSS variables (purple, visible on dark bg) | shadcn default |
