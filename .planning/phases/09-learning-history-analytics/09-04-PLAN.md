2. Create `mystiqor-build/src/app/api/analytics/route.ts`:
   - GET handler with auth check.
   - Parse query params with AnalyticsQuerySchema.
   - Calculate date cutoff based on period (7d=7 days ago, 30d=30 days, 90d=90 days, all=no cutoff).
   - Run 3 parallel Supabase queries:
     a. **Analyses:** Select `tool_type, created_at` from `analyses` where `user_id = user.id` and `created_at >= cutoff`. No limit (personal data volume is small).
     b. **Mood entries:** Select `mood_score, energy_level, created_at` from `mood_entries` where `user_id = user.id` and `created_at >= cutoff`.
     c. **Goals:** Select `status, category, progress, created_at` from `goals` where `user_id = user.id`.
   - Use Promise.all for parallel execution.
   - Aggregate server-side:
     - `toolDistribution`: Record<string, number> — count by tool_type.
     - `activityByDate`: Array<{ date: string; count: number }> — analyses grouped by date (YYYY-MM-DD).
     - `moodTrend`: Array<{ date: string; mood: number; energy: number }> — mood entries by date.
     - `stats`: { totalAnalyses: number; avgMood: number; avgEnergy: number; totalGoals: number; completedGoals: number; goalCompletionRate: number }.
   - Return `{ data: { toolDistribution, activityByDate, moodTrend, stats } }`.
   - Type all response shapes explicitly (no `any`).
