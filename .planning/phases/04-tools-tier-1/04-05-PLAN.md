  <action>
    Create `birth-chart/route.ts` with a POST handler following the established 5-step pattern:

    1. **Auth check:** `createClient()` + `getUser()`, return 401 if not authenticated

    2. **Zod validate input:**
       ```typescript
       const InputSchema = z.object({
         birthDate: z.string().min(1, 'חובה'),  // ISO date string
         birthTime: z.string().min(1, 'חובה'),  // HH:mm format
         latitude: z.number(),
         longitude: z.number(),
         fullName: z.string().min(1, 'חובה'),
       })
       ```

    3. **Planet approximation via LLM** (Pitfall 2 workaround — no ephemeris in Phase 4):
       - Call `invokeLLM` with a system prompt asking for approximate planet positions:
         "You are an expert astrologer. Given the birth date {date}, birth time {time}, and location lat={lat} lon={lon}, estimate the ecliptic longitude (0-360 degrees) for each of these 10 planets: Sun, Moon, Mercury, Venus, Mars, Jupiter, Saturn, Uranus, Neptune, Pluto. Return ONLY a JSON object mapping planet name (lowercase) to longitude number. Example: {\"sun\": 45.2, \"moon\": 123.5, ...}"
       - Use Zod response validation: `z.record(z.string(), z.number())` to parse LLM JSON
       - Convert to `PlanetPositions` format: `{ sun: { longitude: 45.2 }, moon: { longitude: 123.5 }, ... }`
       - If LLM fails validation: fall back to a simple calculation for Sun only (using Julian Date from solar-return.ts), leave other planets at default positions spread evenly (0, 30, 60, ... degrees)

    4. **Chart assembly:**
       - Parse birth datetime: `new Date(`${birthDate}T${birthTime}:00`)`
       - Call `assembleChart(datetime, latitude, longitude, planets)` from `@/services/astrology/chart`
       - This returns `ChartData` with houses + aspects

    5. **AI interpretation:**
       - Read `buildInterpretationPrompt` from `@/services/astrology/prompts/interpretation`
       - Call it with chart data and birth info to build the prompt string
       - Call `invokeLLM` with the prompt for Hebrew interpretation. maxTokens: 1500
       - Parse interpretation text

    6. **Build response with planet details:**
       - For each planet in positions: derive sign (using `getSign(longitude)`) and house (find which house cusp range it falls in)
       - Build planet details array: `{ name, longitude, sign, house, degree_in_sign, is_retrograde: false }`
       - `degree_in_sign = longitude % 30`

    7. **Save to analyses table:** tool_type='astrology', data: { planets, chartData }, result: { interpretation }

    8. **Return:** `{ data: { chartData, planets, planetDetails, interpretation, isApproximate: true } }`

    IMPORTANT: Add `isApproximate: true` flag to response so UI can show disclaimer. Error handling: try/catch, return 500 with Hebrew error.
  </action>
