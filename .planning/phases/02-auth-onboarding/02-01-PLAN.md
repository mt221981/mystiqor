In `mystiqor-build/src/lib/supabase/middleware.ts`, make the following changes:

1. **Replace the dead path check** (line 42) with a working PROTECTED_PATHS array. Delete:
```typescript
if (!user && request.nextUrl.pathname.startsWith('/(auth)')) {
```
Replace with:
```typescript
const PROTECTED_PATHS = ['/dashboard', '/tools', '/onboarding', '/profile', '/settings', '/subscription', '/coach', '/goals', '/mood', '/journal'];
const isProtected = PROTECTED_PATHS.some(p => request.nextUrl.pathname.startsWith(p));
if (!user && isProtected) {
```
Define PROTECTED_PATHS as a `const` array OUTSIDE the function, at module scope, right after the imports. Add a JSDoc: `/** נתיבים מוגנים שדורשים אימות — מידלוור מפנה ל-login אם לא מחובר */`.

2. **Pass `?next=` param** in the redirect URL so login page knows where to send the user back:
```typescript
if (!user && isProtected) {
  const url = request.nextUrl.clone();
  url.pathname = '/login';
  url.searchParams.set('next', request.nextUrl.pathname);
  return NextResponse.redirect(url);
}
```

3. **Inject x-pathname header** into the request for downstream server components (needed by auth layout for onboarding guard). Add this BEFORE the `return supabaseResponse` at the end of the function:
```typescript
const requestHeaders = new Headers(request.headers);
requestHeaders.set('x-pathname', request.nextUrl.pathname);
supabaseResponse = NextResponse.next({
  request: { headers: requestHeaders },
});
// Re-apply cookies to new response
cookiesToSet.forEach(({ name, value, options }) =>
  supabaseResponse.cookies.set(name, value, options)
);
```

IMPORTANT: The cookie syncing is tricky. The current code uses `setAll` callback to update cookies on `supabaseResponse`. After creating a new NextResponse.next for headers, cookies must be re-applied. The cleanest approach: collect cookies set by Supabase in a local array, then apply them to the final response. Modify the `setAll` callback to also push to a local `let pendingCookies: Array<{name: string; value: string; options?: Record<string, unknown>}> = []` array. Then at the end, create the final response with headers AND apply pending cookies.

Alternatively, the simpler approach that preserves existing cookie behavior: inject the header into the existing supabaseResponse rather than creating a new one:
```typescript
supabaseResponse.headers.set('x-pathname', request.nextUrl.pathname);
```
This is simpler and correct -- use this approach. Response headers set via `supabaseResponse.headers.set()` are readable by downstream server components via `headers()` from `next/headers`.

Add Hebrew JSDoc to the function explaining the three responsibilities: session refresh, protected route redirect, x-pathname injection.
