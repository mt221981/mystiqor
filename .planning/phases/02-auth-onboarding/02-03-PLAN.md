**A. Update `mystiqor-build/src/app/(auth)/layout.tsx`:**

Add the onboarding_completed check AFTER the existing auth check. The layout currently:
1. Gets user via `supabase.auth.getUser()`
2. Redirects to `/login` if no user
3. Renders `AuthLayoutClient`

Add between step 2 and step 3:

```typescript
import { headers } from 'next/headers';
```
(add to existing imports at top)

After the `if (!user) { redirect('/login'); }` block, add:

```typescript
// בדיקת השלמת onboarding — מפנה ל-/onboarding אם לא הושלם
const headersList = await headers();
const pathname = headersList.get('x-pathname') ?? '';

if (pathname !== '/onboarding') {
  const { data: profile } = await supabase
    .from('profiles')
    .select('onboarding_completed')
    .eq('id', user.id)
    .maybeSingle();

  if (!profile?.onboarding_completed) {
    redirect('/onboarding');
  }
}
```

Key details:
- The `pathname !== '/onboarding'` check prevents redirect loop (onboarding page is inside the (auth) group)
- `x-pathname` is injected by the middleware (Plan 02-01)
- `maybeSingle()` returns null if no profile row exists — which correctly triggers the redirect
- Use `profile?.onboarding_completed` with optional chaining — handles both null profile and false/null value

Update the JSDoc comment at the top to mention: "בודק אימות, בודק השלמת onboarding, ומפנה בהתאם".
