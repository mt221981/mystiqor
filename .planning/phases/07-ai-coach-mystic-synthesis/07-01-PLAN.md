    **1. Create `mystiqor-build/src/services/coach/context-builder.ts`:**

    Export `buildCoachingContext(userId: string, supabase: SupabaseClient<Database>): Promise<string>`.

    Implementation:
    - Parallel fetch with `Promise.all`:
      - `supabase.from('profiles').select('full_name, birth_date, birth_time, gender').eq('id', userId).maybeSingle()`
      - `supabase.from('analyses').select('tool_type, results, summary, created_at').eq('user_id', userId).order('created_at', { ascending: false }).limit(20)`
      - `supabase.from('goals').select('title, status, category, progress').eq('user_id', userId).eq('status', 'active').limit(10)`
      - `supabase.from('mood_entries').select('mood_score, energy_level, created_at').eq('user_id', userId).order('created_at', { ascending: false }).limit(14)`
    - Build compact Hebrew context string targeting under 2000 tokens:
      - Section `### פרטי המשתמש`: name, birth_date, gender
      - Section `### ניתוחים אחרונים`: for each analysis, extract tool_type + summary only (NOT full results JSONB). For astrology, extract sun/moon/rising from results if present. For numerology, extract life_path from results if present.
      - Section `### יעדים פעילים`: title + status + progress for each
      - Section `### מגמת מצב רוח`: average mood_score over 14 days + trend direction
    - Handle missing data gracefully: if any fetch returns null or empty, skip that section
    - JSDoc in Hebrew. Type the function return as `Promise<string>`.

    **2. Create `mystiqor-build/src/app/api/coach/conversations/route.ts`:**

    Two handlers:

    `GET` — list user's conversations:
    - Auth check via `createClient()` + `supabase.auth.getUser()`
    - `supabase.from('conversations').select('*').eq('user_id', user.id).order('last_message_at', { ascending: false }).limit(20)`
    - Return `NextResponse.json({ data: conversations })`

    `POST` — create new conversation:
    - Auth check
    - Accept optional `{ title?: string }` from request body (default title: `'שיחה חדשה'`)
    - Call `buildCoachingContext(user.id, supabase)` to build initial context
    - Insert into conversations: `{ user_id: user.id, title, context: { text: contextString }, message_count: 0, last_message_at: new Date().toISOString() }`
    - Return `NextResponse.json({ data: conversation }, { status: 201 })`

    **3. Create `mystiqor-build/src/app/api/coach/messages/route.ts`:**

    `POST` — send message and get AI reply:
    - Auth check
    - Validate input with Zod: `z.object({ conversation_id: z.string().uuid(), message: z.string().min(1).max(5000) })`
    - Fetch the conversation: `supabase.from('conversations').select('context').eq('id', conversationId).eq('user_id', user.id).single()` — return 404 if not found
    - Save user message: `supabase.from('coaching_messages').insert({ user_id: user.id, conversation_id: conversationId, role: 'user', content: message })`
    - Fetch prior messages (limit 20 most recent): `supabase.from('coaching_messages').select('role, content').eq('conversation_id', conversationId).order('created_at', { ascending: true }).limit(20)`
    - Build system prompt: start with fixed Hebrew coaching persona string:
      ```
      אתה מאמן אישי מיסטי שמשלב חוכמה עתיקה עם פסיכולוגיה מודרנית.
      אתה מכיר את המשתמש באופן אישי ומשתמש בנתונים שלו כדי לתת עצות מותאמות אישית.
      ענה בעברית בלבד. היה חם, אמפתי ומעשי. השתמש בידע מהניתוחים של המשתמש.
      ```
      Then append the context from `conversation.context.text` (the pre-built coaching context).
      Then append conversation history as a formatted string: `\n\n### היסטוריית שיחה:\n` + messages mapped to `role === 'user' ? 'משתמש' : 'מאמן'` + `: ` + content, joined by `\n`.
    - Call `invokeLLM` with NO responseSchema, NO zodSchema (free-form Hebrew text mode):
      ```typescript
      const response = await invokeLLM({
        prompt: message,
        systemPrompt: fullSystemPrompt,
        maxTokens: 2048,
        userId: user.id,
      })
      ```
    - Extract reply text: `const replyText = typeof response.data === 'string' ? response.data : String(response.data)`
    - Save assistant reply: `supabase.from('coaching_messages').insert({ user_id: user.id, conversation_id: conversationId, role: 'assistant', content: replyText })`
    - Update conversation metadata: `supabase.from('conversations').update({ last_message_at: new Date().toISOString(), message_count: (prior messages count + 2) }).eq('id', conversationId)`
    - Return `NextResponse.json({ data: { role: 'assistant', content: replyText } })`
    - Wrap entire handler in try/catch, return 500 with Hebrew error message on failure

    **4. Add GET handler to `/api/coach/messages/route.ts`:**

    `GET` — fetch messages for a conversation:
    - Auth check
    - Parse `conversation_id` from `request.nextUrl.searchParams.get('conversation_id')`
    - Validate UUID format
    - Verify conversation belongs to user: `supabase.from('conversations').select('id').eq('id', conversationId).eq('user_id', user.id).single()` — return 404 if not found
    - Fetch messages: `supabase.from('coaching_messages').select('id, role, content, created_at').eq('conversation_id', conversationId).order('created_at', { ascending: true })`
    - Return `NextResponse.json({ data: messages })`

    All three files: TypeScript strict, JSDoc in Hebrew, `import type` for type-only imports.
