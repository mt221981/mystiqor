## Spacing Scale

Declared values (all multiples of 4 — 8-point scale):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gap in StandardSectionHeader (`gap-1`) |
| sm | 8px | Icon wrapper internal padding, header description top margin |
| md | 16px | Default element spacing, section horizontal padding |
| lg | 24px | Section vertical padding, header bottom margin (`mb-6`) |
| xl | 32px | Tool page container top padding (`py-8`) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:

| Exception | Value | Reason |
|-----------|-------|--------|
| Icon wrapper in StandardSectionHeader | 44px × 44px (w-11 h-11) | Meets WCAG touch target minimum; existing PageHeader uses 40px — bumped for atmospheric weight |
| Icon size inside wrapper | 24px (w-6 h-6) | Larger than existing 20px to match header hierarchy importance |

This animation preset is added to `src/lib/animations/presets.ts` as `pageEntry`:

```typescript
pageEntry: {
  initial: { opacity: 0, y: 12 },
  animate: { opacity: 1, y: 0 },
  exit: { opacity: 0, y: -8 },
  transition: { duration: 0.6, ease: 'easeOut' },
}
```

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Icon wrapper aria | Icon-only wrapper in StandardSectionHeader is decorative — `aria-hidden="true"` on icon | Screen reader reads h1 title |
| Reduced motion | `useReducedMotion()` applied in StandardSectionHeader and page-level wrappers; CSS `prefers-reduced-motion` removes `result-heading-glow` filter | CONTRAST-03 baseline |

## New CSS in globals.css

Two additions to `@layer utilities`:

```css
/* זוהר כותרת תוצאות — h2/h3 בפלט ניתוח AI */
.result-heading-glow h2,
.result-heading-glow h3 {
  filter:
    drop-shadow(0 0 8px rgba(221, 184, 255, 0.5))
    drop-shadow(0 0 16px rgba(212, 168, 83, 0.3));
}

@media (prefers-reduced-motion: reduce) {
  .result-heading-glow h2,
  .result-heading-glow h3 {
    filter: none;
  }
}
```

Note: Class is `.result-heading-glow` — applied to the wrapper `<div>`, not directly to headings. This allows markdown renderers to generate headings naturally without needing to know about the class.

## Anti-Patterns (executor must avoid)

| Anti-Pattern | Correct Approach |
|--------------|-----------------|
| Using `text-shadow` CSS property for heading glow | Use `filter: drop-shadow(...)` — works on text and is better supported in modern browsers |
| Applying glow directly to markdown `h2`/`h3` elements via inline style | Apply `.result-heading-glow` class to wrapper `<div>` — CSS child selectors handle headings |
| Replacing all `PageHeader` usages including non-tool pages | Only replace in `(auth)/tools/*` pages — dashboard, coach, profile keep `PageHeader` |
| Adding `pageEntry` animation inside `LayoutClient` | Animation must be per-page, not per-layout — layout stays static |
| Hardcoding loading phrases inline in each page | Import from `@/lib/constants/mystic-loading-phrases` — single source of truth |
| Using `left`/`right` for icon positioning in StandardSectionHeader | Use `start`/`end` Tailwind utilities (RTL support) |
| `font-semibold` (600) weight | Only `font-bold` (700) for headings and `font-normal` (400) for body — 2 weights maximum |
| Animating result glow on mount (CSS keyframe flash) | Static CSS `filter` — always-on when element is rendered; no flash animation |
| `useEffect` for delayed icon glow | Use CSS `transition` with `opacity`/`box-shadow` — apply `celestial-glow` class after first render via `useState(false)` + `useEffect(() => setState(true), [])` |
| Importing lucide-react icons for new tool page headers | react-icons/gi only for tool header icons (ATMOS-03) |
