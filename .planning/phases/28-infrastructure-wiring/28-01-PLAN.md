Create `src/app/api/cron/reset-usage/route.ts` implementing a GET handler that:

1. **Auth check** (FIRST thing): Read `authorization` header. Compare against `Bearer ${process.env.CRON_SECRET}`. Return `NextResponse.json({ error: 'Unauthorized' }, { status: 401 })` if missing or wrong. This prevents unauthorized callers from resetting usage counters.

2. **Call DB function**: Use `createAdminClient()` (NOT createClient — no user session in cron). Call `supabase.rpc('reset_monthly_usage')`. This PostgreSQL function already exists and resets `analyses_used` to 0 for all subscriptions.

3. **Error handling**: If RPC returns an error, log it and return 500 with Hebrew error message. On success, return `NextResponse.json({ success: true, reset_at: new Date().toISOString() })`.

4. **Idempotency**: The `reset_monthly_usage()` function is naturally idempotent (sets counters to 0, not decrement). Double-firing on the 1st of the month is safe. No additional idempotency guard needed.

5. **Export maxDuration**: Add `export const maxDuration = 30` (30 seconds is more than enough for a single RPC call).

Follow the project patterns:
- Hebrew JSDoc on exported function and file header
- TypeScript strict (no `any`)
- Import `NextRequest` and `NextResponse` from `next/server`
- Import `createAdminClient` from `@/lib/supabase/admin`
- Hebrew console.log messages for monitoring
- try/catch wrapping entire handler

File should be ~40-50 lines max.
