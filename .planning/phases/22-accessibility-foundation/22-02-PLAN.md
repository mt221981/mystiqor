<objective>
Audit and fix all low-opacity text classes across the codebase to meet WCAG AA contrast ratio (4.5:1) on dark backgrounds. Raise opacity modifiers from /50, /60, /70 to /80 or /85 for meaningful text, preserving decorative opacity where appropriate.

Purpose: Users reported that light text "disappears" into the dark cosmic background. The root cause is Tailwind opacity modifiers (/50, /60, /70) applied to text color tokens, which drops contrast below WCAG AA (4.5:1) even when the base token passes. This is the single most impactful accessibility fix in v1.3.

Output: 38 component files with opacity-corrected text classes. Zero base token changes in tailwind.config.ts or globals.css.
</objective>

<!-- Token values from tailwind.config.ts that define the colors being opacity-modified -->

From tailwind.config.ts:
- `on-surface-variant: '#c8bede'` — base contrast on #0d0b1e is ~6.5:1 (passes AA). At /60 drops to ~3.5:1 (FAILS). At /80 = ~5.2:1 (passes).
- `gold-dim: '#b8913e'` — base contrast on #0d0b1e is ~4.2:1. At /60 = ~2.5:1 (FAILS). At /85 = ~3.6:1 (marginal for large text). Use `text-gold` (#d4a853, ~5.8:1) for critical labels.
- `muted-foreground: hsl(268 20% 78%)` = ~#c4b8d8 — base contrast on muted bg (#242040) is ~4.1:1. At /60 = ~2.5:1 (FAILS).
- `white` (#ffffff) — base contrast on #0d0b1e is 18.8:1. At /50 = ~9.4:1 (passes). At /60 = ~11.3:1 (passes). At /70 = ~13.2:1 (passes). White opacities are generally fine but /50 can be marginal on non-pure-dark backgrounds like glass-panel rgba(14,12,35,0.8).

Opacity fix rules (per D-01, D-02):
- text-on-surface-variant/50 -> text-on-surface-variant/80
- text-on-surface-variant/60 -> text-on-surface-variant/80
- text-on-surface-variant/70 -> text-on-surface-variant/85
- text-gold-dim/60 -> text-gold-dim/85 (or text-gold for important labels)
- text-gold-dim/70 -> text-gold-dim/85
- text-muted-foreground/50 -> text-muted-foreground/80
- text-muted-foreground/60 -> text-muted-foreground/80
- text-muted-foreground/70 -> text-muted-foreground/85
- text-white/50 -> text-white/70 (only if on glass/gradient bg, otherwise keep)
- text-white/60 -> text-white/70
- text-white/70 -> keep as-is (passes AA on pure dark)
- text-error/60 -> text-error/80
- text-primary/60 -> text-primary/80
- text-secondary/60 -> text-secondary/80
- text-tertiary/60 -> text-tertiary/80

EXCEPTION: Border and background opacity modifiers (border-*/50, bg-*/60) are NOT text contrast issues. Do NOT change them. Only change classes that start with `text-`.

EXCEPTION: Hover states (hover:text-*/60) are transient interactions. Fix them to /80 too, but they are lower priority than static text.
