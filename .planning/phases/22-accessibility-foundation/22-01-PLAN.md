<objective>
Define the global z-index scale as CSS custom properties and add prefers-reduced-motion support to all existing animations, then migrate Header, MobileNav, and InstallPrompt to use the new z-index variables instead of hardcoded z-50.

Purpose: Phase 23 adds floating coach (--z-floating: 60) and bottom tabs (--z-tabs: 40). Without a centralized z-index scale defined here, those components will hardcode z-values that collide with Header (currently z-50) and MobileNav (currently z-50). The reduced-motion block prevents WCAG 2.3.3 failures for users with vestibular disorders.

Output: globals.css with z-index scale + reduced-motion block; Header.tsx, MobileNav.tsx, InstallPrompt.tsx using CSS variable z-indexes.
</objective>

**2. prefers-reduced-motion media query (per D-05)**

Add a new block AFTER the `@media print` block (after line 335), at the end of the file:

```css
/* === נגישות: כיבוי אנימציות למשתמשים שמבקשים reduced-motion === */
@media (prefers-reduced-motion: reduce) {
  .stars-bg,
  .aurora-bg,
  .sparkle-float,
  .sparkle-float::before,
  .shimmer-border,
  .shimmer-border::after {
    animation: none !important;
  }

  .mystic-hover {
    transition: none !important;
  }

  .mystic-hover:hover {
    transform: none !important;
  }

  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

This block:
- Explicitly disables the 5 utility classes that use @keyframes (sparkle-drift, aurora-shift, stars-twinkle, shimmer-rotate, sparkle-drift again for ::before)
- Disables mystic-hover transition and transform
- Uses the wildcard selector as a catch-all to kill ALL animation-duration and transition-duration across the entire app
- Preserves scroll-behavior as auto (no smooth scroll)

Do NOT modify any existing CSS rules, @keyframes, or utility classes. This is purely additive.
