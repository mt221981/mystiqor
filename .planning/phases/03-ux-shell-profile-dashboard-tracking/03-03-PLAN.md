Create two API route files following the established pattern.

**File 1: `mystiqor-build/src/app/api/journal/route.ts`**

POST handler:
1. Auth check via `supabase.auth.getUser()` — 401 if no user
2. Validate with `JournalCreateSchema.safeParse(body)` — 400 on failure
3. Insert: `supabase.from('journal_entries').insert({ ...parsed.data, user_id: user.id }).select().single()`
4. Return 201 with data, 500 on DB error
5. Wrap in try/catch

GET handler:
1. Auth check
2. Accept optional query params: `page` (default 1), `limit` (default 20)
3. Calculate offset: `(page - 1) * limit`
4. Query: `supabase.from('journal_entries').select('*', { count: 'exact' }).eq('user_id', user.id).order('created_at', { ascending: false }).range(offset, offset + limit - 1)`
5. Return `{ data, total: count, page, limit }`

**File 2: `mystiqor-build/src/app/api/journal/[id]/route.ts`**

PATCH handler:
1. Auth check
2. Extract `id` from params (Next.js 15: `const { id } = await params`)
3. Validate with `JournalUpdateSchema.safeParse(body)` — 400 on failure
4. Update: `supabase.from('journal_entries').update({ ...parsed.data, updated_at: new Date().toISOString() }).eq('id', id).eq('user_id', user.id).select().single()`
5. Return 200 with data, 404 if no match

DELETE handler:
1. Auth check
2. Extract `id` from params
3. Delete: `supabase.from('journal_entries').delete().eq('id', id).eq('user_id', user.id)`
4. Return 200 on success, 404 if no rows deleted

All error messages in Hebrew. JSDoc comments in Hebrew.
