Create two API route files following the established pattern from analysis/route.ts.

**File 1: `mystiqor-build/src/app/api/mood/route.ts`**

POST handler:
1. `const supabase = await createClient()` (from `@/lib/supabase/server`)
2. Auth check: `const { data: { user } } = await supabase.auth.getUser()` — return 401 if no user
3. Parse body with `MoodCreateSchema.safeParse(body)` — return 400 with `parsed.error.flatten()` on failure
4. Insert into `mood_entries` table: `supabase.from('mood_entries').insert({ ...parsed.data, user_id: user.id }).select().single()`
5. Return 201 with data on success, 500 with error.message on DB error
6. Wrap everything in try/catch — return 500 on unexpected errors

GET handler:
1. Auth check same as POST
2. Accept optional query param `period` (values: `daily`, `weekly`, `monthly`) — default `weekly`
3. Calculate date range using date-fns: `import { subDays, subMonths } from 'date-fns'`
   - daily = subDays(now, 1)
   - weekly = subDays(now, 7)
   - monthly = subMonths(now, 1)
4. Query: `supabase.from('mood_entries').select('*').eq('user_id', user.id).gte('created_at', startDate.toISOString()).order('created_at', { ascending: false })`
5. Return data array

**File 2: `mystiqor-build/src/app/api/mood/[id]/route.ts`**

DELETE handler:
1. Auth check
2. Extract `id` from `params` (Next.js 15 pattern: `params` is a Promise — `const { id } = await params`)
3. Delete: `supabase.from('mood_entries').delete().eq('id', id).eq('user_id', user.id)` (user_id guard prevents deleting others' entries)
4. Return 200 on success, 404 if no rows affected, 500 on error

All Hebrew error messages. All JSDoc comments in Hebrew.
