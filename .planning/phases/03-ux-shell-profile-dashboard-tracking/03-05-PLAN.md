POST handler — WITH SUBSCRIPTION LIMIT ENFORCEMENT (D-12, Pitfall 6):

2. Validate body — create a GuestProfileSchema inline or import a minimal schema:
   ```typescript
   const GuestProfileSchema = z.object({
     full_name: z.string().min(2).max(100),
     birth_date: z.string().regex(/^\d{4}-\d{2}-\d{2}$/),
     birth_time: z.string().regex(/^([01]\d|2[0-3]):([0-5]\d)$/).optional().or(z.literal('')),
     birth_place: z.string().max(200).optional().or(z.literal('')),
     relationship: z.string().max(100).optional(),
   });
   ```
3. ENFORCE LIMIT (per D-12):
   - Count current guest profiles: `supabase.from('guest_profiles').select('id', { count: 'exact' }).eq('user_id', user.id)`
   - Look up subscription: `supabase.from('subscriptions').select('plan_type, guest_profiles_limit').eq('user_id', user.id).single()`
   - Define limits: `const GUEST_LIMITS: Record<string, number> = { free: 1, basic: 3, premium: 8 }`

4. Insert: `supabase.from('guest_profiles').insert({ ...parsed.data, user_id: user.id }).select().single()`
5. Return 201

GET handler:
1. Auth check
2. List all guest profiles for user: `supabase.from('guest_profiles').select('*').eq('user_id', user.id).order('created_at', { ascending: false })`
