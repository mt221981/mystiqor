  The component has two render branches:
  - `if (onSelect)` → renders a `<div role="button">` (comparison mode)
  - else → renders `<Link>` (browse mode)

  In the **Link/browse mode** branch, wrap both the Link and the action buttons in a container `<div>`:
  ```typescript
  return (
    <div className="flex flex-col gap-2">
      <Link href={`/history?id=${analysis.id}`} className="block">
        {cardContent}
      </Link>
      {/* כפתורי ייצוא ושיתוף — רק במצב עיון, לא בבחירה להשוואה */}
      <div className="flex gap-2 flex-wrap px-1" dir="rtl">
        <ExportButton
          toolType={analysis.tool_type}
          summary={analysis.summary}
          results={{}}
          createdAt={analysis.created_at}
        />
        <SharePanel
          analysisId={analysis.id}
          title={`ניתוח ${TOOL_NAMES[analysis.tool_type] ?? analysis.tool_type}`}
        />
      </div>
    </div>
  );
  ```

  **Why `results={{}}`:** `AnalysisCardData` only carries `summary` (not full results JSON) to keep the history list query lightweight.   **Why only in Link mode:** When `onSelect` is set, the card is in comparison selection mode — adding action buttons would conflict with the click-to-select interaction. Export/Share only make sense when browsing.

  **Do not change:** The `cardContent` variable, the `AnalysisCardData` interface, the `AnalysisCardProps` interface, the `onSelect` branch, or any existing styles.
