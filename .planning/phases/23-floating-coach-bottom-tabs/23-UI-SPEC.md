## Spacing Scale

Declared values (all multiples of 4 — 8-point scale):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap in tab bar (`gap-0.5`) |
| sm | 8px | Compact internal padding, tab horizontal padding (`px-2`) |
| md | 16px | Default element spacing, bubble `start-4` offset |
| lg | 24px | Panel section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:

| Exception | Value | Reason |
|-----------|-------|--------|
| Floating bubble size | 56px (w-14 h-14) | CONTEXT.md D-04: touch target comfortably exceeds 44px WCAG minimum |
| Bottom tab bar height | 56px | Matches bubble size; standard mobile nav height |
| Mobile main content bottom padding | 80px (pb-20) | Clearance for 56px tab bar + safe-area-inset-bottom |
| Bubble bottom offset on mobile | 80px (bottom-20) | Sits above tab bar with visual breathing room |
| Bubble bottom offset on desktop | 24px (bottom-6) | No tab bar present |
| Panel height (mobile) | ~380px | CONTEXT.md D-07 |
| Panel width (desktop) | 384px (w-96) | RESEARCH.md Pattern 3 |
| Panel desktop bottom offset | 80px (md:bottom-20) | Above bubble on desktop |

## Interaction States

### Floating Bubble

| Trigger | Visual Response |
|---------|-----------------|
| Idle | Breathing animation (scale 1.0→1.05→1.0, 4s loop) |
| Hover (desktop) | `mystic-hover` lift — translateY(-3px), enhanced glow |
| Press / Click | Panel toggles (open ↔ closed) via `useFloatingCoachStore.toggle()` |
| prefers-reduced-motion | Static — no breathing animation, no hover transform |
| On /coach route | Component returns null — not rendered |

### Chat Panel

| Trigger | Visual Response |
|---------|-----------------|
| Open | Spring slide-up from bottom (mobile) or fade+lift from below (desktop) |
| Close | Reverse animation, unmounted via AnimatePresence |
| First open (empty messages) | Opener message shown as UI-only bubble |
| Sending message | `isLoading: true` → input disabled, spinner visible |
| Error sending | Toast with Hebrew error copy (see Copywriting section) |
| "פתח שיחה מלאה" click | Navigate to `/coach` page |

### Bottom Tab Bar

| Trigger | Visual Response |
|---------|-----------------|
| Tab press | Navigate to href (Next.js `<Link>`) |
| Route change | Active tab updates via `usePathname()` — no animation |
| Active tab | Primary color + glow + indicator dot |
| Inactive tab | Muted foreground color |

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Touch target minimum | 56px bubble + 56px tab bar height — exceeds 44px WCAG minimum |
| Keyboard focus | Bubble: focusable `<motion.button>`; Tabs: `<Link>` elements, naturally focusable |
| Screen reader — bubble | `aria-label="פתח מאמן AI"`, `aria-expanded={isOpen}` |
| Screen reader — tabs | `aria-label="ניווט ראשי"` on `<nav>`, `aria-current="page"` on active `<Link>` |
| Reduced motion | `useReducedMotion()` in FloatingCoachBubble disables breathing animation; panel spring still runs (felt, not seen) |
| Color contrast | Active tab primary `#ddb8ff` on `rgba(10,9,28)` background — passes WCAG AA (Phase 22 baseline verified) |
| RTL | `dir="rtl"` on root layout; `start`/`end` for all positioning; `inset-inline-start/end` for fixed elements |

## Conversation Logic (Claude's Discretion)

On panel first open:
1. Call `fetchConversations()` from `services/coach/api.ts`
2. If conversations exist: set `activeConversationId = conversations[0].id`, call `fetchMessages(id)`
3. If no conversations: `activeConversationId = null`, show opener message only
4. On first user message send with `activeConversationId === null`: call `createConversation()`, then `sendMessage()`

This mirrors the existing flow in `coach/page.tsx handleSendMessage`.

## Anti-Patterns (executor must avoid)

| Anti-Pattern | Correct Approach |
|--------------|-----------------|
| `z-[60]` Tailwind arbitrary z-index | `style={{ zIndex: 'var(--z-floating)' }}` |
| Adding opener message to `messages[]` store | Opener is `const openerMessage = OPENER_MESSAGES[pathname] ?? DEFAULT_OPENER` — rendered as conditional `<div>` only |
| `left`/`right` in CSS or Tailwind | `start`/`end` (Tailwind) or `inset-inline-start`/`inset-inline-end` |
| Sending opener to API | Opener is UI-only, never in fetch payload |
| ReactDOM.createPortal for panel | Not needed — `position: fixed` escapes layout stacking context |
| `persist` middleware on Zustand store | No persistence — state resets on navigation intentionally |
| shadcn/ui Sheet for panel | Use framer-motion AnimatePresence directly (MobileNav comment indicates Sheet not fully configured) |
| CSS `@keyframes` for bubble breathing | Use framer-motion `animate` array + `useReducedMotion()` hook |
