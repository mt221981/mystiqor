**Per D-01 and D-03: Create sync script** at `mystiqor-build/scripts/sync-tarot-meta.ts`.

This script reads `TAROT_CARD_META` and updates/inserts all 78 cards in the `tarot_cards` table. It uses the Supabase service-role key from `.env.local` to bypass RLS for writes.

**Algorithm (per Research Pitfall 1 Рђћ avoid upsert on SERIAL id):**

1. Import `TAROT_CARD_META` from `@/lib/constants/tarot-data`
2. Create Supabase client using `NEXT_PUBLIC_SUPABASE_URL` + `SUPABASE_SERVICE_ROLE_KEY` from process.env (load from `.env.local` using dotenv or manual read)
3. Fetch all existing `tarot_cards` rows from DB
4. Build a Map keyed by `name_en` for existing rows
5. For each entry (cardNumber 0-77) in `TAROT_CARD_META`:
   a. **Determine name_en and name_he:**
      - Major Arcana (cardNumber 0-21): Match to existing row by `number` field. Use existing `name_en` and `name_he`.
      - Court cards (cardNumber 32-35 = wands court, 46-49 = cups court, 60-63 = swords court, 74-77 = pentacles court): Match to existing row by `name_en`. Use existing names.
      - Pip cards (cardNumber 22-31, 36-45, 50-59, 64-73): These are NEW rows. Derive names:
        - `name_en`: Build from suit and rank. Example: cardNumber 22 = "Ace of Wands", 23 = "Two of Wands", etc.
        - `name_he`: Use `archetype` field from TAROT_CARD_META (e.g., 'ОћОЕОеОљОћ' for Ace of Wands, 'ОЕОаОЎОЎОЮ ОЉОъОўОЋОф' for Two of Wands)
   b. **Determine arcana and suit:**
      - Major Arcana (0-21): `arcana: 'major'`, `suit: null`
      - Minor Arcana (22-77): `arcana: 'minor'`, `suit`: derive from card number range:
        - 22-35: 'wands'
        - 36-49: 'cups'
        - 50-63: 'swords'
        - 64-77: 'pentacles'
   c. **Determine numerology_value (Claude's Discretion Рђћ per traditional tarot numerology):**
      - Major Arcana: `numerology_value = cardNumber` (0-21)
      - Pip cards (Ace-10): `numerology_value = rank` where Ace=1, Two=2, ..., Ten=10
      - Court cards: `numerology_value = null` (no standard numerology)
   d. **Build row data** with all fields: name_en, name_he, arcana, suit, number (=cardNumber for major, rank for pip, null for court), meaning_upright (use first 2 uprightKeywords joined as sentence if new row, preserve existing if update), meaning_reversed (use first 2 reversedKeywords joined, preserve existing if update), keywords (=uprightKeywords as array), element, astrology, kabbalah (null if undefined), archetype, upright_keywords, reversed_keywords, numerology_value
   e. **If row exists in DB** (matched by name_en): UPDATE only the 7 new meta fields. Do NOT overwrite name_he, meaning_upright, meaning_reversed, keywords (preserve existing human-authored content).
   f. **If row is new** (pip cards): INSERT complete row.

6. Log results: `Updated: X, Inserted: Y, Errors: Z`

**Card number to suit/rank mapping:**
- For a minor arcana card with cardNumber C:
  - suitIndex = Math.floor((C - 22) / 14) -> 0=wands, 1=cups, 2=swords, 3=pentacles
  - rankInSuit = (C - 22) % 14 -> 0-9 = pip (Ace-10), 10-13 = court (Page, Knight, Queen, King)
