## Critical Integration Gaps (6 Issues)

### Severity 1: Sidebar Tool Navigation Broken
**Impact:** Users cannot reach ANY tool from the sidebar (12 links produce 404)
**Root cause:** Sidebar NAV_SECTIONS hrefs use `/numerology` instead of `/tools/numerology`
**Affected requirements:** All tool requirements (ASTR, NUMR, DRAW, GRPH, TOOL-*)

### Severity 2: Post-Onboarding 404
**Impact:** New users land on 404 after completing onboarding
**Root cause:** OnboardingWizard redirects to `/tools` which has no `page.tsx`
**Affected requirements:** ONBD-03

### Severity 3: Subscription Limits Not Enforced
**Impact:** Free users can run unlimited analyses
**Root cause:** `incrementUsage()` from `useSubscription` hook is exported but never called by any tool
**Affected requirements:** INFRA-03, SUBS-04

### Severity 4: Export/Share Orphaned
**Impact:** Users cannot export to PDF or share analyses — no UI renders these components
**Root cause:** `ExportButton` and `SharePanel` created but never imported into any page
**Affected requirements:** EXPO-01, EXPO-02

### Severity 5: Astrology Sub-Tools Unreachable
**Impact:** Transits, Solar Return, Synastry, Forecast, Calendar pages exist but no navigation path
**Root cause:** Not added to sidebar; main astrology page doesn't link to sub-tools
**Affected requirements:** ASTR-03, ASTR-04, ASTR-05, ASTR-06, ASTR-07

### Severity 6: Sidebar Usage Bar Placeholder
**Impact:** Users see static 42% usage instead of real subscription data
**Root cause:** `PLACEHOLDER_USAGE_PERCENT = 42` hardcoded in Sidebar.tsx
**Affected requirements:** SUBS-04

## Recommendations

The 6 integration gaps are all **wiring fixes** — the underlying features work, they just aren't connected. A single gap-closure phase can address all of them:

1. Fix Sidebar hrefs (add `/tools/` prefix to 12 links)
2. Create `/tools/page.tsx` or change onboarding redirect to `/dashboard`
3. Wire `incrementUsage()` calls into tool page success handlers
4. Render `ExportButton` + `SharePanel` in tool result sections
5. Add astrology sub-tool navigation (sidebar or astrology page internal nav)
6. Replace hardcoded usage bar with live `useSubscription()` data
7. Update REQUIREMENTS.md traceability table (20 items still "Pending" but code exists)
